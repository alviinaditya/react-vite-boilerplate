Production-oriented React boilerplate built with TypeScript, Vite, TanStack Router, TanStack Query, Tailwind CSS v4, shadcn/ui, i18n, strict linting, and Vitest.

## Tech Stack

- React 19 + Vite 8
- TypeScript 6 with strict compiler options
- TanStack Router with file-based routes and generated route tree
- TanStack Query with typed query options and devtools in development
- Tailwind CSS v4 with shadcn/ui preset and design tokens
- i18next + react-i18next + browser language detection
- next-themes for light, dark, and system theme
- react-hook-form + Zod for forms and validation
- ky for HTTP client setup
- Zustand for client state
- Recharts with shadcn chart wrapper
- Vitest + Testing Library
- ESLint, Prettier, Husky, lint-staged, and Commitlint

## Getting Started

Install dependencies:

```bash
npm install
```

Start development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview production build:

```bash
npm run preview
```

## Scripts

```bash
npm run dev           # Start Vite dev server
npm run build         # Generate routes, typecheck, and build
npm run preview       # Preview production build
npm run routegen      # Generate TanStack Router route tree
npm run typecheck     # Generate routes and run TypeScript checks
npm run lint          # Run ESLint with zero warnings
npm run format        # Format project files
npm run format:check  # Check formatting
npm run test          # Run Vitest in watch mode
npm run test:run      # Run Vitest once
```

## Project Structure

```text
src/
  app/
    providers/        # App-level providers
    routes/           # TanStack Router file routes
    router.tsx        # Router setup and router type registration
  components/ui/      # shadcn/ui components
  config/             # Typed app configuration
  features/
    dashboard/        # Example feature module
    i18n/             # Translation setup and locale files
    theme/            # Theme switcher feature
  lib/                # shadcn utilities
  shared/
    api/              # Shared HTTP client and API helpers
    lib/              # Shared utilities
  test/               # Test setup
```

Feature modules should keep their own API, model, UI components, and tests close together.

## Routing

Routes live in `src/app/routes`.

TanStack Router generates `src/route-tree.gen.ts`. Do not edit this file by hand. It is regenerated by:

```bash
npm run routegen
```

`npm run build` and `npm run typecheck` run route generation automatically before TypeScript checks.

## Server State

Use TanStack Query for server state.

Prefer colocated query option factories:

```ts
export function dashboardSummaryQueryOptions() {
  return queryOptions({
    queryKey: ['dashboard', 'summary'],
    queryFn: fetchDashboardSummary,
    staleTime: 1000 * 60 * 5,
  })
}
```

Route loaders should call `queryClient.ensureQueryData(...)` when route data should be prefetched.

## UI And Styling

This project uses Tailwind CSS v4 and shadcn/ui.

Add shadcn components with:

```bash
npx shadcn@latest add button
```

Components are configured to live in:

```text
src/components/ui
```

Use semantic tokens such as `bg-background`, `text-foreground`, `text-muted-foreground`, `border-border`, and shadcn component variants before adding custom styling.

## Internationalization

i18n is initialized in `src/features/i18n/i18n.ts`.

Translations are split by language and namespace:

```text
src/features/i18n/locales/en/common.json
src/features/i18n/locales/en/dashboard.json
src/features/i18n/locales/id/common.json
src/features/i18n/locales/id/dashboard.json
```

Use namespaces in components:

```ts
const { t } = useTranslation('dashboard')
```

Language detection uses `i18next-browser-languagedetector` and stores the selected language in `localStorage` under `app-language`.

## Code Quality

TypeScript and ESLint are intentionally strict.

Important rules include:

- No explicit `any`
- No non-null assertions
- No unsafe assignment/member access/return
- Exhaustive switch checking
- Consistent type imports
- Zero lint warnings in CI-style checks

Generated files and shadcn registry components have targeted lint exceptions where third-party typing requires it.

## Git Hooks

Husky and lint-staged are configured.

On pre-commit:

- Prettier formats staged project files
- ESLint checks staged TypeScript files

Commit messages use Conventional Commits via Commitlint.

If this project has not been initialized as a Git repository yet, run:

```bash
git init
npm run prepare
```
